Standard Notes on Memos

Financial Year & Numbering

The financial year (e.g. 2026-27, running 1 April to 31 March) is detected automatically from the memo/receipt date. Set the numbering series prefix and next number for the current financial year below. You can edit these whenever a new financial year begins.
All data is stored on this device by default. Use Sync below to keep it up to date across your iPhone, iPad and Mac.

Cross-Device Sync (iCloud / Google Drive)

Every time you add or edit a memo or receipt, this app saves automatically on this device — no separate "save" step needed. To carry your data across devices, use the buttons below.
On iPad / iPhone / Mac (any browser): tap "Save Backup to Cloud". The share sheet opens — choose Google Drive, iCloud Drive, Files, Dropbox or OneDrive, and save the backup there (use the same folder each time so it keeps the latest copy). On your other device, open the app and tap "Load Backup from Cloud" (or "Restore from Cloud Backup" on the Home screen), then pick that file from the same Google Drive / iCloud folder. The load merges in anything new, so you can do it freely without creating duplicates.
On Mac (Safari 15.4+, Chrome or Edge) only: you can also tap "Link Synced File" and choose a file inside iCloud Drive or Google Drive (in Finder) for hands-free live syncing — every change writes to that file and is pulled back in on open.

Import from Excel / CSV

Import memos and/or receipts in bulk from a spreadsheet (.xlsx or .csv). Download a template first to see the expected columns, fill it in, then import. Existing memos/receipts are matched by Memo No. / Receipt No. — matching rows update existing records, new numbers create new records.

Import Historical Records ("All Memos Subdivided")

Import directly from the "all_memos_subdivided.xlsx" register — the sheet with separate Date, Court/Tribunal, Solicitor, Matter No., Parties, Work Done and Fees columns. Each row becomes a historical memo entry, with its Financial Year worked out automatically from its date. Rows are matched to existing memos by financial year + serial number, so re-importing the same file never creates duplicates.
If a very large file fails to save fully in one go (your browser's storage is full), use this to bring the file in one year at a time instead — repeat with a different year each time.
If dates look wrong or missing from a previous import, use "Delete & Re-import" below — this removes every previously-imported historical memo first, then lets you pick the file again for a clean import. Memos you created manually in this app are never touched by this.

Data & Multi-Year

Export creates a complete backup (all memos, receipts and settings across all years). Import can either replace all data, or merge a backup from another device/year into this one without losing existing records.

Backup — Hard Drive & Cloud

Hard drive (local): "Backup Now to Device" saves a dated JSON snapshot of everything (memos, receipts, contacts, settings) to your Downloads / Files folder. Keep copies on your Mac's hard drive periodically.
Cloud: the Cross-Device Sync above already keeps a live copy of your data inside iCloud Drive once linked, which iCloud also backs up online. For an additional manual cloud backup, use "Backup Now to Device" and then drag the downloaded file into iCloud Drive, Google Drive, OneDrive or Dropbox.

WhatsApp Contacts

People you have WhatsApp'd memos or reminders to are remembered here, so you can reuse their numbers. Add, edit or remove them below.
Import contacts from your phone / WhatsApp: WhatsApp and the Contacts app can export contacts as a .vcf (vCard) file — open a contact (or select several), choose Share / Export, and save the .vcf to Files or email it to yourself. You can also import a simple .csv with Name and Phone columns. Pick that file below to bring those names and numbers in here.

Danger Zone

These actions permanently erase data from this device and cannot be undone. Take a backup first (above) if there is any chance you'll want this data again. This does not affect data already saved to iCloud Drive in previous syncs unless you sync again afterwards.
